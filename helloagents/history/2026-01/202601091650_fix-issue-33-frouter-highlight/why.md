# 变更提案: 修复 issue #33（浅色主题 FRouter 选中高亮）

## 需求背景

issue #33 反馈：暗色主题下被选中的 FRouter 会被高亮，便于识别当前选中项；但浅色主题下高亮缺失或显示异常，导致无法快速确认当前选中的 FRouter。

期望：浅色主题下选中态高亮调整为“黑色边框”。

## 变更内容

1. 修复浅色主题中 FRouter 列表选中态样式未生效/异常的问题
2. 将浅色主题下选中态高亮改为黑色边框（保持暗色主题不变）

## 影响范围

- **模块:** 前端 UI / 主题（`frontend/theme/`）
- **文件:** `frontend/theme/light.html`
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 浅色主题下突出当前选中的 FRouter
**模块:** 前端 UI（主题）

#### 场景: 选中 FRouter 后识别当前项
用户在 FRouter 列表中点击某个 FRouter 后：
- 被选中的 FRouter 行呈现黑色边框高亮
- 未选中的行保持原样
- 暗色主题表现不受影响

## 风险评估

- **风险:** 主题 CSS 变量兼容/样式调整可能影响少量组件观感
- **缓解:** 仅补齐浅色主题缺失的兼容变量，并将改动范围限定在 FRouter 列表相关样式；通过手动切换浅色/暗色主题验证选中态与 hover 态
