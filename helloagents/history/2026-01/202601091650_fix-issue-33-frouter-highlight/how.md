# 技术设计: 修复 issue #33（浅色主题 FRouter 选中高亮）

## 技术方案

### 核心技术
- **前端:** 纯 HTML/CSS（`frontend/theme/light.html`）

### 实现要点
- 在浅色主题 `:root` 中补齐/映射被 UI 样式引用但未定义的 CSS 变量（用于兼容现有样式写法）
- 将 `.node-row.active` 的选中态高亮改为黑色边框，并避免使用与浅色主题不匹配的固定色值
- 暗色主题不做变更，确保现有高亮效果保持不变

## API设计

无（纯前端样式调整）

## 数据模型

无

## 安全与性能

- **安全:** 无用户输入处理与权限变更
- **性能:** 仅 CSS 变量与样式调整，不引入额外运行时开销

## 测试与部署

- **手动验证:** 切换浅色主题，点击不同 FRouter，确认选中态为黑色边框；切换暗色主题确认无回归
- **工程验证:** 执行 `go test ./...`（确保仓库整体无编译/单测回归）
